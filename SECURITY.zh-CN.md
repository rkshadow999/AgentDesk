# 安全策略

[English](SECURITY.md) | [简体中文](SECURITY.zh-CN.md)

## Alpha 支持边界

AgentDesk 仍处于 Alpha 阶段。安全修复会应用到当前 `main` 分支，以及存在正式版本时最新的受支持版本。旧快照、未签名 CI 包、本地修改版和第三方重新打包版本可能不会获得修复。

当前 `NativeProtected` 配置被明确标记为本机兼容模式，而不是内核级沙箱。因此，按照当前 Windows 用户的文件系统与网络权限进行预期访问，本身不构成漏洞。绕过权限决定、凭据边界、风险警告、进程清理、沙箱证明或文档声明的 fail-closed 行为，则属于安全问题范围。

`WslStrict` 当前不可用，因为系统尚不能证明全部子进程都受到网络限制。如果严格证明失败后仍发生执行、认证、静默降级或会话创建，应将其视为安全问题并私下报告。

## 私下报告漏洞

请使用 GitHub Private Vulnerability Reporting：

https://github.com/rkshadow999/AgentDesk/security/advisories/new

不要在公开 Issue、Discussion、Pull Request 或提交中包含尚未披露的漏洞、利用代码、凭据、私有提示词、个人信息或私有源码。如果 GitHub Private Vulnerability Reporting 暂时不可用，请稍后重试，不要公开敏感细节。

请只提供调查所需的信息：

- 受影响的提交、版本、包类型、架构和执行配置。
- 安全影响，以及被跨越的边界。
- 最小复现步骤或经过清理的小型复现项目。
- 相关日志，并移除 API Key、Token、用户名、本地路径、提示词和私有源码。
- 问题是否已被利用，或是否已在其他地方披露。

## 后续处理

仓库维护者会通过私有安全公告验证报告、协调修复、在适当时申请 CVE，并商定披露方式。处理时间取决于严重程度、可复现性、受影响的上游代码和发布准备情况。在商定协调披露日期，或维护者确认可以公开之前，请不要披露问题。

如果报告影响继承的 `xai-org/grok-build` 代码，AgentDesk 维护者可能会与相关上游维护者私下协调。只有在修复确有需要时，才会共享报告者信息和私有附件。

## 范围

安全报告可涉及 WinUI/WebView2 宿主、ACP 传输与 sidecar 生命周期、Rust AgentDesk 扩展、凭据存储、权限处理、执行配置强制措施、WSL 证明、打包、更新或发布元数据以及仓库自动化。如果漏洞位于独立依赖中，也应向该依赖的维护者报告；当它会实质影响 AgentDesk 时，请附上依赖的安全公告引用。

基于仓库证据的[威胁模型](docs/AGENTDESK-THREAT-MODEL.zh-CN.md)记录了预期本机权限、信任边界、滥用路径与残余风险。该文档的限制不会缩小上述私有报告范围。
